写这个博客的时候，也就是今天，距离我学完mysql数据库已有3天左右。但是，还没有将所学到的东西转化为自己的东西，将之应用于自己的生活中，让它成为我技术的一部分。

下面，是我所学到的基本东西。

#1.Myssql 初涉
1.1Mysql数据库的概念理解。它是一个关系型的数据库，与redis相比，它是存在硬盘里的，而redis是将数据存到缓存里。mysql更适合存大量的基本数据，redis读取数据的速度非常的快，因此，常常用来存一些热门数据。

1.2Mysql数据库服务的启动与停止:’NET START MYSQL’
NET STOP MYSQL

1.3Mysql数据库的登录与退出 登录：mysql -uroot -p -P3306 -h127.0.0.1
参数说明：-p password -P 端口号 -h服务器地址（如果是连接远程的mysql服务，需要连接远程的服务器地址）退出 \q;

1.4Mysql提示符的设置四种提示符：\D(完整的日期) \d(当前数据库) \u(当前的用户) \h(当前的服务器名称) 改数据库提示符 PROMPT \u@\h \d> 改成了 root@localhost 当前数据库名

1.5数据库的基本操作 数据库的创建+字符集的修改 CREATE DATABASE t1 CHARACTER SET gbk(utf8); 查看当前创建的所有数据库 SHOW DATABASES; 查看某一个数据库的基本信息 SHOW CREATE DATABASE t1;

#2.数据类型与数据表的操作

##2.1 数据类型

3.日期时间类型：YEAR TIME DATE DATETIME TIMESTAMP
4.字符型：VARCHAR(M) 0<M<=255 TEXT ENUM(‘VALUE1’,’VALUE2’,’VALUE3’)

读书分享会

Posted on 2019-03-24 | In 阅读

这是我第一次在参加完读书分享会之后进行总结，可能文字表达会比较生疏。

读书分享会：分享会是<<如何阅读一本书>>中的一个观点开始的：读书应该不借助任何的外力来让自己读懂。然后梁安旋学妹对作者的这个观点表示怀疑，认为如果没有任何外力的帮助，很有可能花了大量的时间仍旧不能理解作者的观点，反而如果借助其他的工具，或者询问其他人，会比较迅速的理解到作者的意思。我们对这个话题进行了讨论，最后得出的结论是，对于不同的书籍应该采用不同的方法对待。大家讨论的热情被这个话题慢慢地带动了起来。之后，黄祺轩学弟为我们讲述了<<如何阅读一本书>>的概要，用这本书中的话，就是完成了读书的第一步。在这里，我意识到我和学弟，学妹读的是两本不同的书，就是书的名字一样但是内容架构上有所不同。但是很奇妙的是两本书所表达的内容和方法论是极其相似的。之后，我们的话题就止不住了，从出售时间到财富自由，从问老妪打仗的问题到思维导图，范围很广，但是都是围绕着如何做好一件事情而展开的。

能和一起爱读书，喜欢学习的人在一起讨论书籍，本身就是一件很快乐的事。(ps,下次分享会少说点，多多聆听 ）
