这是我第一次以月为单位对算法进行回馈，总结
经历了一个月的学习，保证每天都至少有一个小时来学习算法，最后，发现了一些自己在算法学习上的感悟，总结，以及改进。

回顾这一个月的历程，最后剩下的是什么东西：两样：1.具体的经典排序算法的思想 2.从算法思想到具体代码实现的解决问题的能力的缓慢提升。

这两个东西都是十分的重要，关于 基本的算法思想，毫无疑问，是需要记忆的，是需要写在本子上反复记忆揣摩的，要用记忆规律来将之形成神经突触。关于具体的代码实现，这个是无法记忆的，也是没有必要记忆的，记忆反而会让你感觉很沉重，让你 解决问题的能力下降，对于这一点，要十分的铭记。但是，当你做了很多类型的题目，在这些题目之间找到代码实现的一些共性的技巧的时候，这个是非常宝贵的，是需要写成博客进行记忆的。

在一开始的时候，为了让自己在月底反馈的时候，可以让自己很安慰地觉得自己学了很多东西，就利用记忆规律记很多东西（算法思想，具体实现，实现的细节），但这明显让自己抓不住重点，而且对解决问题能力的提升是致命的伤害。（因为你在遇到新问题的时候不能 就题论题，当然，基本的算法思想让你如虎添翼）

因此，在此，我决定对自己学习算法的方法进行以下几点的改进。
1.需要保持的一点，也是最重要的是：在知道了算法思想后，自己不看代码实现， 立刻（如果没有紧急情况的话）独立完成代码实现，这个是对提升解决问题的能力来说最重要的一点。
2.关于 复习基本的算法思想 。 对于某个问题的算法思想（注意，只有算法思想，没有实现），要利用记忆规律，（1,2,3,15,31）来进行突触的建立。这个工作是在本子上完成的。
3.关于 博客 的书写。博客的书写一个月一次（不能再中间写），博客上写什么东西，博客上写的东西一定是经过了自己这一个月的思考，而形成的思想的精华（意味着很少，但是很干货），不会在学习资料上直接看到的东西。（例如，和之前算法的对比，关于代码实现的一些技巧性的东西）
4.每个月的 月底复盘博客。

###归并排序
归并排序递归算法思想：对于两组有序的数据，我们可以对之进行合并并排好序。因此，我们可以利用递归， 将数据进行拆分，每次2分，直到拆到只有一个数据，一个数据本身就是有序的，然后再归并，归并的过程中调用我们写好的对两组有序的数据进行归并并排序的算法。当递归归并完成时，就排好序了。

 void merge(int [] a,int L,int mid,int R) {//将a[L,mid] 和 a[mid+1,R]两个有序的数组进行归并 前闭后闭
    int [] copy= new int[R-L+1];//归并排序，每次进行两个有序数组的合并，都要开辟一个新的数组，将数组中的值copy一份 
    for(int i=L;i<=R;i++) {//copy[]数组索引是从0开始的
        copy[i-L]=a[i];
    }
    int i=L,j=mid+1;//声明三个索引，分别指向copy数组的当前比较的值和原来数组的遍历索引
    for( int k=L;k<=R;k++) {
        if(i>mid) {//数组越界
            a[k]=copy[j-L];   //！！！数组越界问题出现在这个位置，原来是a[k]=copy[j],越界了，现在改为copy[i-L],好了！！！！
            j++;
        }else if(j>R){//数组越界的判断
            a[k]=copy[i-L];
            i++;
        }
        else if(copy[i-L]<copy[j-L]) {//注意这个位置的数据的比较，copy[i-l]和copy[j-l]而不是copy[i]和copy[j]
            a[k]=copy[i-L];
            i++;
        }else  {
            a[k]=copy[j-L];
            j++;
        }
    }


}

}

1.归并排序每次进行merge操作的时候，要进行额外的空间的开辟。这也是归并排序的缺点之一。
2.创建的数组的索引和原数组的索引差了L个偏移量。在对copy数组元素进行比较的时候，注意是比较的copy[i-L] copy[j-L]
3.数组越界问题的处理（容易忽略) i>mid 和 j>R 的处理。
4.边界问题的处理。 边界问题很关键，往往能决定你的代码能否写成功。首先，定义你的索引边界，是闭区间还是开区间，然后去维护你的索引，这点很关键。 对取到和取不到进行考虑。

###快速排序(20世纪最伟大的算法之一)

####1.单路快速排序
快速排序的思想 将数据的第一个位置插入到合适的位置，使得它前面的元素都是小于它的，他后面的元素都是大于它的，相当于把数组分成3部分，<v,=v,>v，然后进行递归，就完成了整个数组的排序。如何将第一个元素放到合适的位置，并对数组完成了分组呢？这个是快速排序的核心(partition)。方法：索引的建立， j索引 (>v,<v的分界点)， i索引(遍历索引)，temp值(数组第一个位置元素的值)对i进行遍历，如果a[i]>temp,不做变化，如果a[i]<temp,交换a[i]和>v的数组的第一个元素的值，即a[j+1],然后j++。最后，遍历结束后，交换a[j]和a[l],返回j索引，对a[l,p-1] 和 a[p+1,r]递归排序。

public class QuickSort {
SortTestHelper sth=new SortTestHelper();
void quickSort(int [] a) {
    quickSortSon(a,0,a.length-1);//递归函数的构造  递归的本质是将相同的问题不断地缩小规模，然后逐个解决
    //sth.printArray(a);
}
void quickSortSon(int [] a,int l,int r) {//对数组a[l,r]左闭合右闭合进行排序       ！！！注意对递归函数的构建，对个人来说也是一个难点！！！！
    if(l>=r)
        return;//递归终止条件

    int p=getIndexOfBound(a,l,r);//返回一个索引，在这函数执行完之后，满足a[l,p-1]<a[p]   a[p,r]>a[p]
    quickSortSon(a,l,p-1);//傻逼的错误！！！！！ 之前 quickSort(a,0,p-1) 应该改为 quickSort(a,l,p-1)
    quickSortSon(a,p+1,r);
}
int getIndexOfBound(int [] a,int l,int r) {
    int d=sth.createRandomNum(l,r);//数组最左边的索引  (快速排序的优化就发生在partition的过程中，改进之前，每次考虑的元素为数组的最左边的元素，改进之后，用随机数的索引作为考虑的元素)
    int mid=0;//这样做对于近乎有序的数据的排序有十分大的改进，经过数学证明，算法的期望值为n*log(n),退化为n*n级别的算法概率几乎为0
    mid=a[d];
    a[d]=a[l];
    a[l]=mid;
    int temp=a[l];
    int j=l;//分界索引 注意j的起始位置，试了之后发现从l开始比较好
    int transfer=0;
    for(int i=l+1;i<=r;i++) {//i的起始位置为l+1!!!! 注意=号要加上！！！！！！（不加就错了）因为是闭区间，如果不加=，就取不到最后一位
        if(a[i]<temp) {//如果后一个元素比v小,将a[i]和>v数组中的最小元素  a[j+1] 交换位置，同时j++
            transfer=a[i];
            a[i]=a[j+1];
            a[j+1]=transfer;
            j++;
        }
    }
    transfer=a[j];//为什么不能使用swap函数
    a[j]=a[l];
    a[l]=transfer;

    return j;
}

}
注意点
1.partition操作中，j分界索引的建立。

####2.双路快速排序
双路快排算法思想
1.遍历从两边开始遍历，i(左边遍历索引),j(右边遍历索引)索引建立。当i遍历到a[i]>=a[l]结束遍历，j便利到a[j]<=a[l]时结束，然后当两个都遍历结束时，交换两个索引所指向的值，i++,j++。所有遍历结束后，交换a[l],a[j]的值。返回j索引，然后递归遍历a[l,j-1] a[j+1,r]。和单路快排比较的优点，将=v的元素均分到两边，达到一种更趋近于logn 的分级，使得算法时间复杂度更趋近于nlogn.

void doubleQuickSortSon(int [] a,int l,int r) {//对数组a[l,r]左闭右闭进行排序
    if(r-l<=30) {//递归终止条件没有加                                      //？？？？？？为什么不在小范围进行插入排序，直接采用递归的方式会报数组越界的问题？？？？？？？？
        ssq.UpdateInsertSort(a);
        return ;

    int p=partition(a,l,r);//返回边界的索引
    doubleQuickSortSon(a,l,p-1);
    doubleQuickSortSon(a,p+1,r);

int partition(int [] a,int l,int r) {//对单路返回索引的改进，采用从两头开始遍历，这样的好处在于当处理有很多重复的数据时，能更好的将重复的数据平分给两边，从而达到一种logn的分级。
    int j=r;//右边的遍历索引
    int i=l+1;//左边的遍历索引
    int mid=0;//进行优化，随机出来一个数，作为参照值，防止有序数据对数组的退化
    mid=sth.createRandomNum(l, r);
    sth.swap(a[l], a[mid]);
    while(true) {//注意while循环的使用 while循环内部为循环条件
        while(a[i]<a[l]&&i<=r) i++;//数组越界
        while(a[j]>a[l]&&j>=l+1) j--;
        if(i>j) break;
        sth.swap(a[i], a[j]);
        i++;
        j--;
    }
    sth.swap(a[l],a[j]);
    return j;

}

}
注意点
1.while语句的使用，外层为while(true),内层为两个while(),break的使用。
2.数组越界的处理
3.索引的建立与维护。
优化
1.在r-l<15时，用插入排序代替。
2.不以数组的第一个元素作为参照点，而是随机产生一个值作为参照点。目的在于产生更加均衡的分级。

####3.三路快速排序
三路快速排序算法思想 三路快速排序中的三路不是指从三个方向进行遍历，其遍历方向只有一个，从左到右依次遍历。只不过它将数组分为严谨的三部分，<v,=v,>v,对于=v的部分，下次递归的时候不用考虑，就可以一次性不用考虑很多元素。具体实现思路：创建索引，l,r,i(遍历索引),lt(<v的索引最大值),rt(>v的索引最小值).对当前遍历到的元素a[i]进行分类考虑，a[i]>v,a[i]<v,a[i]=v.然后进行相应的操作。遍历完之后，交换a[l]和 a[lt+1],然后对a[l,lt] a[rt,r]进行递归实现。

三路快排代码实现 

public class ThreeWayQuickSort {
SortTestHelper sth=new SortTestHelper();
SortSquare ssq=new SortSquare();
void quickSort(int [] a) {//对数组a进行排序
    quickSortSon(a,0,a.length-1);
    sth.printArray(a);
}
void quickSortSon(int [] a,int l,int r) {//对数组a[l,r]左闭右闭进行递归排序
    if (r-l<15) {
        ssq.UpdateInsertSort(a);
        return ;
    }
    int lt=l;//小于v的数据的右边端点
    int rt=r+1;//大于v的数据的左边端点 注意初始值的选取，选取r+1。[rt,r]>v   [l+1,lt]<v [lt+1,i)=v
    int i=l+1;//左边遍历索引 只需要从一个方向遍历
    int mid=sth.createRandomNum(l, r);
    sth.swap(a[l], a[mid]);
    while(i<rt) {//对于这种需要从多边遍历的，使用while循环，不用for循环更方便
        if(a[i]==a[l]) {
            i++;
        }else if(a[i]<a[l]){
            sth.swap(a[i], a[lt+1]);//交换位置 交换的位置错误！！！！！应该为lt+1的位置
            lt++;
            i++;
        }else if(a[i]>a[l]) {
            sth.swap(a[i], a[rt-1]);//不用维护索引i，因为替换过来的a[i]还没有被处理 ！！！！交换位置错误，应该为rt-1的位置
            rt--;
        }
    }        
    sth.swap(a[l], a[lt+1]);//当遍历完之后，swap一下[l]和 a[lt+1]
    quickSortSon(a,l,lt);
    quickSortSon(a,rt,r);
}                                //关于partition（）有无的问题，返回结果为数组，就不将partition（）单独拉出来写个函数了！！！

}
优点
对于处理重复元素较多的数据，可以达到很快的速度，因为可以过滤掉很多不必考虑的元素。

下面，是我所学到的基本东西。

#1.Myssql 初涉
1.1Mysql数据库的概念理解。它是一个关系型的数据库，与redis相比，它是存在硬盘里的，而redis是将数据存到缓存里。mysql更适合存大量的基本数据，redis读取数据的速度非常的快，因此，常常用来存一些热门数据。

1.2Mysql数据库服务的启动与停止:’NET START MYSQL’
NET STOP MYSQL

1.3Mysql数据库的登录与退出 登录：mysql -uroot -p -P3306 -h127.0.0.1
参数说明：-p password -P 端口号 -h服务器地址（如果是连接远程的mysql服务，需要连接远程的服务器地址）退出 \q;

1.4Mysql提示符的设置四种提示符：\D(完整的日期) \d(当前数据库) \u(当前的用户) \h(当前的服务器名称) 改数据库提示符 PROMPT \u@\h \d> 改成了 root@localhost 当前数据库名

1.5数据库的基本操作 数据库的创建+字符集的修改 CREATE DATABASE t1 CHARACTER SET gbk(utf8); 查看当前创建的所有数据库 SHOW DATABASES; 查看某一个数据库的基本信息 SHOW CREATE DATABASE t1;

读书分享会

Posted on 2019-03-24 | In 阅读

这是我第一次在参加完读书分享会之后进行总结，可能文字表达会比较生疏。

读书分享会：分享会是<<如何阅读一本书>>中的一个观点开始的：读书应该不借助任何的外力来让自己读懂。然后梁安旋学妹对作者的这个观点表示怀疑，认为如果没有任何外力的帮助，很有可能花了大量的时间仍旧不能理解作者的观点，反而如果借助其他的工具，或者询问其他人，会比较迅速的理解到作者的意思。我们对这个话题进行了讨论，最后得出的结论是，对于不同的书籍应该采用不同的方法对待。大家讨论的热情被这个话题慢慢地带动了起来。之后，黄祺轩学弟为我们讲述了<<如何阅读一本书>>的概要，用这本书中的话，就是完成了读书的第一步。在这里，我意识到我和学弟，学妹读的是两本不同的书，就是书的名字一样但是内容架构上有所不同。但是很奇妙的是两本书所表达的内容和方法论是极其相似的。之后，我们的话题就止不住了，从出售时间到财富自由，从问老妪打仗的问题到思维导图，范围很广，但是都是围绕着如何做好一件事情而展开的。

能和一起爱读书，喜欢学习的人在一起讨论书籍，本身就是一件很快乐的事。(ps,下次分享会少说点，多多聆听 ）
