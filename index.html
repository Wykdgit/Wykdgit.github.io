#算法心得
之前多多少少也学了算法的一些知识，只不过学的太急，真的太急，而且没有研究学习算法的方法。导致学习效果不理想。个人认为，算法这个东西真的是急不来的，着急的人，是不可能学好算法的。
个人认为，正确的学习算法的步骤应该是这样的。
1.重视基础，即数据结构。 应该花很多的时间去学习这个东西，最后能达到手写任何一种数据结构，并可以将之应用到实际的场景中，才算及格（自己尚未及格）
2.对于经典的算法。 在学习一个经典算法的时候，要先了解代码的思路，然后自己尝试用代码的方式来实现。这点真的很重要，这也是要锻炼的核心能力。之后应真正清晰， 准确的了解代码每一行的意义，并且很准地写出来。对一些基本的难懂的概念（递归），要清晰地理解，不理解，直到理解为止。
3.关于写题
写题是学习算法的最后一步（目前还未触及），也是成为算法大牛的最重要一步。
写题的重要性
1.写题是将你所学的所有算法内容集合起来，然后去解决实际问题。只有能将所学内融汇贯通，才能写出题目来。
2.写题可以起到复习所学算法知识的作用。
3.写题可以提高编程能力，编程能力的最高境界就是知道解决问题的方法，就可以对之进行代码实现（个人认为）。
如何写题
1.了解题目的问题，运用自己所学的所有算法知识，去解解决问题（一定要坚持2天，2天后写不出来，gg）
2.当自己解决出来了之后，查看别人的结题方法，然后用别人的思路再去解决这个问题，比较优劣势。
3.将别人的可取的技巧，算法思想，加到自己的算法体系中，只有这样，才有提高，才有成长。

现在的我走到了学习算法的第二步，但是感觉自己的第一步没有走的太扎实，所以，在走完第二步的时候，我要回过头去走第1步。

###归并排序
归并排序算法思想：对于两组有序的数据，我们可以对之进行合并并排好序。因此，我们可以利用递归，将数据进行拆分，每次2分，直到拆到只有一个数据，一个数据本身就是有序的，然后再归并，归并的过程中调用我们写好的对两组有序的数据进行归并并排序的算法。当递归归并完成时，就排好序了。
代码实现：

 void mergeSortSon(int[] a,int L,int R) {//将数组a[L,R]排序
    //利用递归来进行拆分
    if(L>=R)//if(R-L>15) InsertSort() return；对于小数组，可以采用插入排序 
        return ;//递归终止条件

    int mid=(L+R)/2;//递归中有3条语句，每个函数必须三条语句都执行完才能出栈       ！！！！递去的过程进行的语句！！！
    mergeSortSon(a,L,mid);
    mergeSortSon(a,mid+1,R);
    //if(a[mid]>a[mid+1])如果满足这个条件才进行merge，否则，不merge
    merge(a,L,mid,R);//将a[L,mid] 和 a[mid+1,R]两个有序的数组进行归并 前闭后闭     ！！！！归来的过程进行的语句！！！

 }


 void merge(int [] a,int L,int mid,int R) {//将a[L,mid] 和 a[mid+1,R]两个有序的数组进行归并 前闭后闭
    int [] copy= new int[R-L+1];//归并排序，每次进行两个有序数组的合并，都要开辟一个新的数组，将数组中的值copy一份 
    for(int i=L;i<=R;i++) {//copy[]数组索引是从0开始的
        copy[i-L]=a[i];
    }
    int i=L,j=mid+1;//声明三个索引，分别指向copy数组的当前比较的值和原来数组的遍历索引
    for( int k=L;k<=R;k++) {
        if(i>mid) {//数组越界
            a[k]=copy[j-L];   //！！！数组越界问题出现在这个位置，原来是a[k]=copy[j],越界了，现在改为copy[i-L],好了！！！！
            j++;
        }else if(j>R){//数组越界的判断
            a[k]=copy[i-L];
            i++;
        }
        else if(copy[i-L]<copy[j-L]) {//注意这个位置的数据的比较，copy[i-l]和copy[j-l]而不是copy[i]和copy[j]
            a[k]=copy[i-L];
            i++;
        }else  {
            a[k]=copy[j-L];
            j++;
        }
    }


}

}
注意点
1.归并排序每次进行merge操作的时候，要进行额外的空间的开辟。这也是归并排序的缺点之一。
2.创建的数组的索引和原数组的索引差了L个偏移量。在对copy数组元素进行比较的时候，注意是比较的copy[i-L] copy[j-L]
3.数组越界问题的处理（容易忽略) i>mid 和 j>R 的处理。
4.边界问题的处理。 边界问题很关键，往往能决定你的代码能否写成功。首先，定义你的索引边界，是闭区间还是开区间，然后去维护你的索引，这点很关键。 对取到和取不到进行考虑。
归并算法的优化
1.当递归到l-r<15的时候，进行插入排序，会有更好的效果。（不知道为啥？？？待处理）

这是我第一次在参加完读书分享会之后进行总结，可能文字表达会比较生疏。

读书分享会：分享会是<<如何阅读一本书>>中的一个观点开始的：读书应该不借助任何的外力来让自己读懂。然后梁安旋学妹对作者的这个观点表示怀疑，认为如果没有任何外力的帮助，很有可能花了大量的时间仍旧不能理解作者的观点，反而如果借助其他的工具，或者询问其他人，会比较迅速的理解到作者的意思。我们对这个话题进行了讨论，最后得出的结论是，对于不同的书籍应该采用不同的方法对待。大家讨论的热情被这个话题慢慢地带动了起来。之后，黄祺轩学弟为我们讲述了<<如何阅读一本书>>的概要，用这本书中的话，就是完成了读书的第一步。在这里，我意识到我和学弟，学妹读的是两本不同的书，就是书的名字一样但是内容架构上有所不同。但是很奇妙的是两本书所表达的内容和方法论是极其相似的。之后，我们的话题就止不住了，从出售时间到财富自由，从问老妪打仗的问题到思维导图，范围很广，但是都是围绕着如何做好一件事情而展开的。
