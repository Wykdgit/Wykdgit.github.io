算法学习第二个月末尾复盘与总结

学了什么

经过两个月的算法学习，终于算是将算法的基本的东西学习和实践了一遍，包括排序算法；堆，索引堆，索引堆的优化（反向查找）；二分查找（查找问题）；二分搜索树，树的建立，二分搜索树的三种遍历方式（深度优先遍历），层序遍历，树中元素的增删改查操作，二分搜索树的局限性问题以及解决方案（红黑树），由二分搜索树引申的树形问题（快排，归排）；并查集，三个版本的并查集以及改进，并查集解决的问题，最终的路径压缩；图论，图的组成（节点和边），两种建立图的方式（邻接矩阵，邻接表），两者的优缺点，遍历邻边操作（最常见的操作，迭代器实现），无权图的两种遍历方式（深度优先遍历，广度优先遍历），无权图的最短路问题（广度优先遍历），有权图，与之相关的最小生成树问题（三种方式，lazy prim，prim，kruscal算法），有权图的最短路径问题（dijkstra算法，无负权边），对于有负权边的算法（bellmanford算法）。

下面，对本月学到的每个工具进行一次重温。

堆

1.堆可以解决什么问题？
主要是两类问题
a.优先队列问题，每次出列优先级最高的那个任务。比如：操作系统的很多调度算法，都会用到堆。
b.N个元素中选出前M个元素的问题。利用堆，可以很高效的的解决。

2.基本的二叉堆的定义
a.最小堆
本质上是一棵完全二叉树，每个节点都小于它的子节点。
b.最大堆
是一棵完全二叉树，每个节点都大于它的子节点。

3.堆的代码建立
由于堆是一棵完全二叉树，因此，可以用数组来实现。
为了方便，数组从1开始存储数据，这样就有简单的数学规律。左子节点：2n 右子节点：2n+1 父节点：n/2

4.堆的基本方法和功能
a.最底层的方法两个，shiftUp(),shiftDown()
b.利用基本的两个方法，来实现insert(),入堆操作，extraxtMax(),出堆操作

5.堆排序
将数组中的数据一个一个insert()堆中，然后执行n次extractMax()操作，存回数组中。

6.原地堆排序
基本的堆排序会开辟额外的数组空间（即堆中的数组data[]）,为了不开辟额外的数组空间，利用原地堆排序
AM:将数组看成是堆，对之进行heapify操作，之后，他就真的变成了堆。之后，将堆的第一个元素和最后一个元素进行调换位置，此时第一个元素放在了正确的位置，为了维护此时第1到n-1个位置认为最大堆，对第一个元素进行shiftDown操作，注意每进行一次，所需维护的堆元素个数减一，不然又将最大的元素推到堆顶。 

void heapSort(int [] a) {//原地堆排序(因为是数组，所以索引从0开始)，不需要开辟额外的空间，直接将数组看成堆，在上面进行排序。
    int mid=0;
    for(int i=(a.length-1)/2;i>=0;i--) {
        shiftDown2(a,a.length,i);//这里，所需要维护的堆的长度是不变的
    }
    for(int i=a.length-1;i>0;i--) {//交换完位置，然后shiftDown  //skill 5：for循环遍历的顺序是不固定的，根据实际情况可以从前往后或者从后往前遍历
        mid=a[0];
        a[0]=a[i];
        a[i]=mid;
        shiftDown2(a,i,0);//这里，所维护的堆的长度是变化的，每次交换一次，就减一
    }
}

7.索引堆
1.为什么使用索引堆？？
因为当某个任务的权重变化的时候，堆不能改变这个任务的权重。而索引堆可以巧妙地解决这个问题。索引堆indexes[],用来存放数组的索引，而数据本身没有任何变化。shiftUp(),shiftDown()的也是索引，而不是数据，但是需要利用数据来进行比较。
2,索引堆在insert()时，需要指定插入的索引和数据。
3.索引堆的最主要方法，change(),传入两个参数，索引，权重，改变某个任务的权重。

8.索引堆的优化（反向查找）
利用索引堆，每次找索引i在索引堆中的顺序的时候，都需要遍历一遍索引堆。indexes[j]==i。可以利用反向查找，开辟一个新的数组空间，reverse[],来存放索引在堆中的位置。
reverse[indexes[i]]=i
indexes[reverse[i]]=i
